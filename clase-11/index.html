Responsive Web Design (RWD)

Es una metodologia de trabajoque busca que el layout como el contenido de una pagina o aplicacion se vea correctamente en cualquier tamañano de viewport..

No existen propiedades especificas de Css especificas para Responsive Web Design por lo tanto RWD es un conjunto de practicas

Hay dos enfoques principales :

  • Mobile First (comenzar diseñando la version movil e ir subiendo el tamaño)
  • Cotent First (tener en cuenta el contenido para aplicar cambios segun el tamaño de pantalla)

Meta Viewport

Es un tipo de meta introducido por Apple que le dice al navegador como interpretar las dimensiones de la pagina respecto a las dimensiones del viewport

El meta viewport no tiene ningun efecto en navegadores de escritorio

<meta name="viewport" content="width=device-width, initial-scale=1.0"> 
		
Media Queries

			
Son reglas (at-rules) que permiten condicionar los estilos siempre que se cumplan
			ciertas condiciones.Inicialmente se usan para identificar los tipos de medios por ejemplo
			(impresora:print,pantalla:screen) pero tambien se usan para identificar el tamaño del viewport
			,orientacion del dispositivo o densidad de pixeles
			

		
			

				@media screen {
					body{
					  background: #000;
					  color: white;
					}
				}
				@media print {
				  body{
					background: #fff;
					color: #333333;
				  }
				}
				@media all {
				  body{
					font-family: 'STARWARS', cursive;
				  }
			}

Las condiciones se unen con el operador and

		
			@media screen and (min-width: 200px){
				  body{
					color: rebeccapurple;
				  }
			}
		
	
	
Las condiciones quieren usar las comas como el operador logico OR

	

	@media print , (min-width: 200px){
	  body{
		color: red;
	  }
	}

Tambien se puede detectar la orientacion del dispositivo :

	

	@media screen and (orientation: landscape) {
		//dispositivo en horizontal
	}

	@media screen and (orientation: portrait) {
		//dispositivo en vertical
	}

BreakPoints

Son medidas relativas al ancho del viewport en las que se cambian el diseño.Existen dos tipos:

  • Major breakpoints (para cambios mayores como layout).Generalmente se definen al inicio del proyecto o desde un framework
  • Minor breakpoints (para cambios pequeños).Se definen sobre el desarrollo del proyecto

Los breakpoints se combinan con las media queries.

Es mas usual construir las media queries usando los breakpoints como min-width e ir subiendo de tamaño

Sass para RWD

Las medias queries se pueden anidar al selector

	

	body {
	  background: red; }
	  @media screen and (min-width: 400px) {
		body {
		  background: green; } }
	  @media screen and (min-width: 700px) {
		body {
		  background: yellow; } }
	  @media screen and (min-width: 1000px) {
		body {
		  background: purple; } }

Otra forma en la que Sass ayuda al RWD es que se pueden crear mixins para simplificar las media queries

	
		/*Mixin de ed-grid*/
		@mixin desde($size){
		  @media screen and (min-width: $size) {
			@content;
		  }
		}
		@mixin hasta($size){
		  @media screen and (max-width: $size) {
			@content;
		  }
		}
		@mixin desde-hasta($desde,$hasta){
		  @media screen and (min-width: $desde) and (max-width: $hasta){
			@content;
		  }
		}

Resolucion vs Viewport

La resolucion es la cantidad de pixeles fisicos que hay en una pantalla(ancho x alto) mientras que el viewport es el tamaño real que existe para diseñar , generalemente ambos conceptos coincidian.Pero todo cambia a partir del IPhone4 y el Retina Display
